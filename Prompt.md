# TTNS 优化

本仓库进行 TTNS 优化相关的科学研究，目标如下：

+ 使用 TTNS 近似高维密度函数。
+ 使模型能够处理多层 DAG 密度近似（当前实现以树结构为阶段性目标）。

你需要为后续 Agent 编写交接文档 `Program.md`（**仅由人类维护和优化**）。文档中应描述：当前 TTNS 模型、必做事项、当前进度，以及 Agent 继续工作所需的全部信息。

+ 在**所有迭代**中，必须验证 TTNS 模型的正确性。

+ 当前问题：在多层 TTNS 进行传播时，计算缓慢，结果效果差，怀疑有 bug。

## 语言与文档规则（强制）

1. **所有面向人类的回答必须使用中文**（聊天回复、问题解释、方案讨论、结论总结）。
2. **所有新建或更新的项目文档正文必须使用中文**，包括但不限于：
   - `Program.md`（人类维护，Agent 除非被要求否则不得修改）
   - `simple_ttns_l2/reports/` 下的实验报告 markdown
   - 本仓库内其他说明性 `.md` 文件
3. **以下保持英文或原文**，不必翻译：
   - 源代码、标识符、API 名称、CLI 参数、文件路径
   - 终端命令与代码块
   - 数学符号与公式
   - 论文标题、外部链接、数据集专有名称
4. **Markdown 中的数学公式统一使用 `$` 定界**（不使用 `\(` `\)`、`\[` `\]`）：
   - 行内公式：`$...$`，例如 `$q = \tilde{q}^2$`
   - 行间公式：`$$...$$`，公式独占一行或多行
5. Agent 在 Cursor 中应遵循 `.cursor/rules/chinese-language.mdc` 中的持久规则。
6. 若用户明确要求使用其他语言，以用户当次指令为准。
