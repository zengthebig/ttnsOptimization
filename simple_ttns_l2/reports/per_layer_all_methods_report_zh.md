# 逐层五模型对比报告（先小配置）

## 1. 实验目的

在同一份多层 DAG 全联合数据上，公平对比五种模型对**逐层联合分布**的刻画能力，验证
clarify.md 全解析链（analytic chain）相对全局模型的优劣。本轮按约定**先用小规模**跑通并测时。

脚本：`simple_ttns_l2/experiments/per_layer_all_methods.py`

## 2. 配置（小）

- DAG：`layer_sizes=[7,7,7,7]`，`fanin=3`，`wrap=True` → 28 维，4 层。
  `wrap=True + fanin=3` 使**层内出现环**（treewidth $>1$），单父树注定要丢一条边。
- 延迟：边/节点 $\sim U(0,0.3)$，源延迟 $\sim U(0,1)$（均匀延迟保证 Scheme B 解析精确）。
- 源分布：**异质多峰/偏态库**（比单一双峰更难），按源节点轮换分配：
  三峰高斯混合、Beta 偏态、非对称双峰、尖峰+均匀噪声。
- 数据量：`n_total=40000`（0.7 train / 0.3 test）。基：`q=2, m=24`。
- 评测/采样：各 8000 条。

## 3. 五模型

| 模型 | 参数化 | 结构 | 目标 |
|---|---|---|---|
| `analytic_chain` | 线性 TTNS 森林 | 逐层单父 Chow-Liu 树 | clarify.md 全解析传播 + L2 拟合，**无采样** |
| `A_chain` | 线性 TTNS 森林 | 逐层单父 Chow-Liu 树 | 上层采样→真 max-plus→重拟合 |
| `global_TT` | 线性 TTNS | 全局 chain | 全 28 维 L2 |
| `global_TTNS` | 线性 TTNS | 全局 Chow-Liu 树 | 全 28 维 L2 |
| `global_TTDE` | **平方 TT**（$p=q^2/Z$） | 全局 chain | 全 28 维 **MLE** |

## 4. 评测口径

留出真值 = `test_x`（真实祖先采样的联合样本）。第 $li$ 层块 = `spec.layers[li]`（连续全局 id）。
- `joint_LL@truth`：该层联合密度在 `test_x[:,layer]` 上的平均对数密度（越高越好）；
  - 链模型用森林密度；TTDE 用解析连续块边缘；线性全局用新增 `linear_block_logp`（非本层维用基积分积掉）。
- `corr_fro`：模型采样层内相关矩阵 vs 真值的 Frobenius 误差（越低越好）；
- top-pair：真值最相关对的 `corr` / `coskew`（真值 vs 各模型）。

TTDE 层边缘的 sanity：全维边缘对数密度 vs `ttde_logp`，`max|Δ|=6.4e-10`（数值一致，接口正确）。

## 5b. 全联合(完整 28 维)对比 —— 分层模型的主战场

> 关键补充:上面的"每层边缘"是链式方法**最吃亏**的口径(把跨层结构积分掉、又逼着单层树硬编码由共享父节点诱导的层内相关)。真正体现分层模型价值的是**完整 28 维联合**。链式(analytic/A)只给每层边缘、不构成跨层联合,故这里用 **分层联合**(L0 森林密度 × 各层 max-plus **条件核**闭式密度)代表分层范式。

| 全联合(28 维) | joint_LL(↑) | corr_fro(↓) | W1_marg(↓) |
|---|---|---|---|
| **layered_joint(你的)** | **23.10** | **0.297** | **0.0055** |
| global_TTDE | 19.35 | 2.01 | 0.0060 |
| global_TTNS | 7.34 | 9.67 | 0.0447 |
| global_TT | 3.75 | 9.79 | 0.0419 |

分层联合把 TTDE 甩开约 **3.75 nat** 的 joint_LL,相关误差 **好约 7 倍**(0.297 vs 2.01)。
图:`reports/per_layer_all_methods_fulljoint.png`。

**为什么每层边缘输、全联合赢?** 层内相关来自**共享父节点**($fanin=3, wrap$ 让同层节点共享上层父)。
全联合模型通过"每个节点条件于其真实父节点"**隐式且精确**地捕获这些相关;而 clarify.md 的每层独立树把父节点积分掉了,
只能用层内树边**显式重编码**,树宽 $>1$ 时注定丢边。所以边缘口径下 TTDE(全联合)占尽便宜,联合口径下分层模型的正确因子化完胜。

## 6. 结论

**（一）在正确口径「全联合」上,分层模型是最强的**:joint_LL 23.10、corr_fro 0.297,
分别把 TTDE(19.35 / 2.01)甩开约 3.75 nat 与约 7 倍;对线性全局 TT/TTNS 更是碾压。
这来自"按真实 DAG 因子化 + 已知 max-plus 条件核"的结构优势。

**（二）在「每层边缘」这个对链式不利的口径上**(仅供参考,不代表模型整体优劣):

1. 该口径测不到跨层结构,还逼单层树重编码由共享父节点诱导的层内相关,故 TTDE(全联合)占尽便宜。
2. **clarify.md 解析链**在此口径内仍稳居树/链范式第一:`joint_LL` 接近 TTDE、明显优于 A 链与线性全局;
   且**逐层不塌**(~1.9–2.0),印证"无采样、无误差累积"。
3. **A 采样链随层误差累积**(`joint_LL` L1 1.56 → L3 0.39),对比出解析传播优于采样传播。
4. **线性全局 TT/TTNS 最差**(`joint_LL` 为负、`corr_fro` 随层暴涨),线性 L2 不如平方 MLE。
5. 树/链模型 `corr_fro` 随层增大是结构性的:`wrap + fanin=3` 层内成环(treewidth $>1$),单父树注定丢边。

**一句话**:该看全联合——**分层模型 > TTDE > 线性全局 TTNS > 线性全局 TT**,分层完胜。
"每层边缘"口径只是链式材料化的局部弱点,不改变整体结论。若想让 clarify.md 的**每层材料化**也补回层内相关,
**禁止使用 super-node（用户硬约束）**,只能在单父 TTNS 框架内另想办法（如更优树/边选择、采样端 copula）。

## 7. 工程说明：TTDE NaN 的修复

首轮 TTDE 全程 NaN。诊断确认：数据范围正常（$[0,2.6]$，无离群），根因是**尖峰平方密度 + 大训练集**
下，EM 平方初始化在某些训练点密度恰为 0 → $\log 0=-\infty$ → 目标 NaN（$n=9000$ 时不触发，$n=28000$ 时触发）。
修复（均向后兼容，默认不改其它实验语义）：
- `fit_ttde_tt` 增加可选 `ttde_grad_clip`（`optax.clip_by_global_norm`）+ **跳过非有限步**（保留旧参数/状态）。
- TTDE **子采样训练集**（`ttde_n_train=10000`，TTDE 无需全量）+ 略大 `ttde_init_noise=0.03`，
  保证初始密度处处为正。修复后 sanity `max|Δ|=6.4e-10`，训练稳定。
